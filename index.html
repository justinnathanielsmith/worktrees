INITIALIZE WORKFLOW
01 / SPEED

Instant Switching

No `git checkout` delays. Branches exist as parallel directories. Switch contexts as fast as you can `cd`.

02 / POWER

Parallel Tasks

Run heavy test suites on `main` while simultaneously developing on `feature-x`. No resource locking.

03 / FOCUS

Clean Roots

The Git engine stays hidden in `.bare`. Your workspace remains focused on the code that matters.

EXECUTION PROTOCOL

PHASE 1: THE ENGINE

Clone your repository as a bare instance into a hidden directory.

git clone --bare git@github.com:user/repo.git .bare
PHASE 2: THE BRAIN

Point the root directory to the hidden engine using a gitdir pointer.

echo "gitdir: ./.bare" > .git
PHASE 3: THE MUSCLE

Spawn independent worktrees for your active branches.

git worktree add main
git worktree add -b feature-dev dev
